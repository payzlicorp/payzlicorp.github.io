Included in Release

This release focuses on many fixes and enhancements across Payzli Connect modules such as partner, support, residuals, risk and other modules.

Dashboards

  • Partner Dashboards enabled for specific users

Residuals

  • Esquire merchant statement job is created.
  • Transaction & Activities page does not load data and appears to be empty is fixed.
  • Renamed Transaction and Activities to Activities
  • Bug Fixes:
  • Date format corrected for -
    • Report summary
    • Billing element tab
    • Detail tab
    • Merchant tab

Risk

  • Route fixes for Triage and Investigation pages.
  • Fixed Scroll in table view.

Multi Merchant - Multi-Location Support for Merchant Accounts

Support for multiple merchant locations under a hierarchy. During lead creation, users can add new locations to existing chains or create new ones. Merchant profiles will show multiple accounts for easy switching. Users can search for existing organizations and add new locations to them. The multi-location flow starts when a new merchant lead is created, allowing seamless integration and management of multiple locations. Users can link and delink locations, and toggle between them to access specific information. This update enhances multi-location management.

Enhancements

  • PDF fields have been improved to accommodate auto wrap feature for text.
  • New permission Manage Attachments (inside Lead Management) is introduced, whoever have this permission can now add attachments irrespective of the account state.
  • Risk, Underwriter and admin roles will see underwriting tab even after account moves to open state

Partner Bug Fixes

  • Sales partner profile not opening through search has been fixed.
  • Hierarchy id issue while creating partner is fixed.
  • Organization name will not be changed now when a new user is added.

Support

  • Merchant account has been handled in terms of permissions -

  • SMS, Internal Notes is limited to partner and internal employees.
  • If a merchant email address is added into the support case recipient list, then that case is accessible to merchant.

  • Recipient list modified to include merchant and partner mail addresses.

  • UI Fixes -
    • Text field expander
    • Time stamp for notes
    • Case details button style

Users And Rights

  • Permissions have been grouped now under categories.
  • UI enhancements to provide elevated user experience.
  • User history log enhancements to track user activities throughout.

User Login

  • Email verification causing interruptions in logins has been improved.
  • Equipment categories population failure in signup flow which caused hiccups is fixed.
  • Login api integration was missing post confirmation flow, it is handled now.

Other enhancements and Fixes

  • Fixed the bug leading to failures while closing an account.
  • Comment screen disappearing for accounts if fixed now.
  • Repay failures have been handled internally to ensure smooth functionality.
  • Convert to Physical app option has been added back in kanban view.
  • 2FA report generation time has been fixed.
  • Empty spaces being shown in Members dropdown has been fixed.
  • Search will now be reset once you come back to merchant screen from visited merchant through global search.
  • Industry display name is changed to Nail Salon.
  • Kanban related UI enhancements have been pushed to provide better user experience.
  • Data precision improvements have been made.
  • Express XML certificates have been renewed.
  • Transaction & Activities page does not load data and appears to be empty is fixed.
  • Underwriting Manager facing issues to edit profile data is fixed.

Quality Assurance

  • The QA team worked on adding more use cases on UI automation and working on the end-to-end testing automation flows

Known Issues and Limitations

  • Merchant will only be able to see support case when added as a recipient or replied to thread from connect portal.
  • For already created partner accounts, where sales experience wasn’t selected will show empty.
  • For Admin, to view all the support cases you need to navigate to Support Module. They will have access to all the cases there.

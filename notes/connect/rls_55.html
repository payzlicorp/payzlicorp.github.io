Included in This Release

1. Merchant Profile

  • New Profile & Onboarding Experience:
    A revamped merchant profile screen and onboarding flow.

  • Merchant Account Status Fix:
    Resolved the issue where a Merchant Account’s status was not marked as completed post-eSign.
    • Direct emails sent through Adobe (third-party) have been turned off.
    • eSign emails will now only work within Connect.

  • Support Cases Visibility:
    Added functionality to view support cases created by merchants or when merchants are added as recipients on the Merchant Profile page.

  • Deleted Merchant Lead Visibility:
    Leads that are deleted from the system will now be fully removed from visibility for all users, avoiding confusion.

2. Step Flow

  • POS to Connect Integration:
    Integrated step-up flow for POS to Connect and created support ticket functionality for deployments.

3. Bug Fixes

  • Residual Program Screen Bug Fixes:
    Addressed issues within the residual program screen.

  • Additional Owner Issue:
    Fixed an issue when adding a second owner.

    • Previously, while in Lead status, adding additional owners would sometimes redirect the user to the “Authorized Representative” field upon interacting with the “Search Address” field for Additional Owner 1.

Known Issues

  • SMS Functionality:
    SMS is currently not functioning for sending Merchant Applications or Support cases.
    This is a work in progress and is dependent on the SMS service provider.

  • Visa Partial Boarding - Approve Button:
    The approve button is disappearing in Visa partial boarding.
    Workaround: Set the responsible flag value to false until the issue is resolved.

  • Adobe eSign Error:
    An issue has been identified where Adobe returns the following error: ```json { “code”: “AGREEMENT_NOT_EXPOSED”, “message”: “The agreement has not yet been exposed to the requested participant” }

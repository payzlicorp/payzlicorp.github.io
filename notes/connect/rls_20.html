Included in Release

Permission Management

  • In the Advanced Search All tab, visibility is now restricted to the organization level, providing a more streamlined user experience.
  • invite user flow should show Organization as a mandatory field
  • Add new member in a team will work properly

Move Incomplete Partners to Open State

Accounts that were previously in incomplete states due to issues with the Invite Partner Link functionality have now been moved to open status.

Attachment Management

  • Attachment size limit has been increased to 20 MB to accommodate larger files.
  • Handled attachment long names
  • Shown document type in support attachments section

Partner Leads and Profile Updates

  • Kanban cards now display Partner First Name and Last Name instead of Company Name
  • Target industry and target zip codes are now optional fields.
  • Business Address - same as Home Address and made it optional
  • In add new lead, mobile number is now optional field
  • Dorpdown option list for changing partner lead stage/status is added in partner profile screen. ‘New Partner Signed’ stage will remain disabled.
  • Driving license number minimum character limit is removed.
  • Handling of incorrect custom PDFs in partner account creation. PDF validations is added to validate pdf format during upload process to ensure no errors are occurring during/ before esign
  • Removed undefined status label from partner leads

Flex Fee

  • Change flex fee order - All flex fees should be on left , and selected flex fees should appear on right side of table
  • Flex Fee Display Names have been updated to remove codes and provide more descriptive names. eg ‘180 – PCI fee’ will be displayed as Annual PCI Fee
  • Fixed issues in updating flex fee values issue

Business Card

  • Show option to create business card even if partner have not completed assessment
  • Business card - changed placing of phone number and email

Filter

  • Improvement in filter by sales partner organization
  • Filter and show equipment’s based on auth platform

History

  • Account approved comment will have time stamp date
  • Prevent Empty logs generation by cron job shown in history

Merchant Profile

  • Resolved issue with choose equipments in lead conversion flow
  • Do not allow changing auth platform for partially boarded merchant as MID is generated based on auth platform.
  • Amex data fields should not populate in Cost Plus
  • Removed Duplicate field - Average Ticket
  • Added retry visa product setup
  • Internal boarding section - added validation message for visa/masteracrd program id

User Interface Improvements

  • Removed time from Business Start Date and Birth dates, only date will be there

Miscellaneous Updates

Numerous bug fixes have been implemented, including: - Office365 email connectivity issue fixed - New support case creation will only be allowed after office 365 connectivity - To avoid duplicate records, same UI action will be blocked till the time previous action is complete

Various improvements have been made to the UI, attachment management, partner lead management, and bug fixes to enhance overall system stability and performance. We believe these updates will significantly improve your experience with Connect. Thank you for your continued support and valuable feedback. If you encounter any issues or have further suggestions, please don’t hesitate to reach out to our tech team.

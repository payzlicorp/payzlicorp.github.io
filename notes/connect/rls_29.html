Included in Release

Features:

  1. Multi-MID Setup Enhancement - Introduced functionality for users to select the organization they wish to log in as when using a single email address associated with multiple applications. - Users can now complete the sign-up process via confirmation links for unsigned applications. - All associated accounts are visible in the login section for ease of access.

  2. New Esquire BIN Adjustments - Added Esquire Bank to the allowed bank list. - Enabled automatic generation of MIDs and sort codes based on Esquire Bank selection for merchants and partners.

  3. Sandbox Environment Access - Implemented a feature allowing merchants to request access to sandbox accounts. - Merchants can now test integration in sandbox environments provided by Visa and Repay.

  4. Show Filtered Users List - Limited users are displayed while creating a new ticket in “Assigned to” and “Members”. - Users profile picture will be displayed for easy identification.

  5. Main Menu Organization - Optimized the layout and organization of the main menu for improved navigation.

  6. Password Reset History Record - Added history record functionality for password reset actions.

Bug Fixes:

  1. Partner Terms PDF Update - Revised PDF documents to comply with partner terms requirements.

  2. Required Fields and Date Display Fix - Set Business Start Date, SSN, and Address as mandatory fields before e-signature. - Resolved issues with incorrect date display in Connect’s View vs. Edit modes for “business_start_date” and “birth day”.

  3. Fix for No Website - Match data pulling and tsys boarding error handling for no website. - Fixed the issue where “https://” was added as default value if no business website was provided in the merchant profile screen.

  4. Pricing Values UX Improvement - Addressed issues related to defaulting to incorrect values during editing.

  5. Partner Communication Fixes - Corrected invalid date and time display in partner communication SMS. - Resolved errors encountered while sending training emails to partners.

  6. Support Case Communication Permission - Implemented error messaging for permission issues, providing clear feedback when users lack access to support inboxes.

  7. Kanban/Table View Selection - Ensured working “Clear Option” for all filters in the sidebar.

  8. Convert to Partner Settings Retrieval - Fixed errors occurring during the retrieval of saved settings when converting to a partner.

  9. Password Issues - Improved error handling for forgot password API responses, providing accurate feedback for invalid emails. - Implemented proper error messages for inactive user accounts during password reset attempts.

  10. Other: - Miscellaneous Bug Fixes: Addressed various other minor bugs and enhancements.

Known Issues:

Esquire Bank MID Discovery: Discover MID functionality for Esquire Bank is currently unavailable and will not be added in this release.

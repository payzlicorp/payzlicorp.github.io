Release Notes: Version 9.0.0

Overview

Version 9.0.0 brings a host of improvements, bug fixes, and UX design changes to enhance the performance and usability of our POS system. This update addresses critical issues, refactors code for improved maintainability.

What’s Changed

UX Enhancements:

  1. Appointments Flow Revamp: The all-new, cutting-edge appointment booking flow offers the simplest way to find available slots for technicians. It includes smart suggestions for upselling to customers based on their previous appointments and a manual override for multiple bookings in the same slots if needed. Additionally, the UX has been enhanced for a smoother and quicker appointment booking experience. The appointment details page carries additional information organized more cohesively to improve the experience of the staff.

  2. Customers Tab: The new customers tab has been completely revamped, offering a smoother user experience for existing functionalities and introducing exciting new features like updated graphs and metrics.

  3. Dashboard: The new dashboard is cleaner with an updated user interface and graphs. This will help increase readability for important metrics and trends as well as view the most important information for both admins and technicians. Sales Tab: The sales tab has been improved with reorganization of information and better user experience.

Bugs Fixes and Code Refactoring:

  1. General Bug Fixes: Addressed various bugs to improve system stability and performance. (#916, #922, #927, #932, #940, #945, #949, #958, #966, #977, #990, #1052, #1060, #1085, #1095, #1102, #1133, #1144, #1155, #1169)
  2. Logout Issue: Fixed an issue causing improper logout functionality. (#938)
  3. Session Expiry: Addressed bugs related to session expiry and ‘Remember Me’ functionality. (#941, #1010)
  4. Mail Invitation Login: Resolved issues with the login flow for mail invitations. (#1019)
  5. Appointment Notes/Comments: Fixed issues with appointment notes and comments. (#1009)
  6. Refund Transactions: Corrected bugs in the refund transaction process. (#1124)
  7. Printer Integration: Fixed standard printer integration issues. (#1151, #1180)
  8. Cash Discount: Resolved bugs related to cash discount functionality. (#1152)
  9. SignalR and Redis: Fixed issues with SignalR classes and Redis server usage. (#1145, #1171)
  10. Flutter SDK: Updated the Flutter SDK version to resolve compatibility issues. (#1174) Refactoring
  11. Code Refactoring: Minor refactoring to improve code readability and maintainability. (#950, #996, #1016, #1028, #1130, #1139, #1175, #1193)

Performance Improvements

  1. POS APIs: Enhanced POS API performance for faster and more reliable operations. (#1056)
  2. Sales and Transaction Refactoring: Refactored sales and transaction processes to enhance performance and reduce errors. (#1075, #1089, #1094, #1096) New events have been added that can be tracked on Amplitude
